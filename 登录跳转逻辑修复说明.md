# 登录跳转逻辑修复说明

修复时间：2026-06-26  
版本：v2.3

---

## ⚠️ 问题说明

### 严重问题
之前我在修改客服聊天页面时，**错误地将所有用户的登录跳转都改成了客服工作台**，导致：
- ❌ 普通用户登录后无法访问主页
- ❌ 普通用户被强制跳转到客服页面
- ❌ 卖卡、交易等主要功能无法使用

**非常抱歉犯了这个严重错误！**

---

## ✅ 修复内容

### 修复前（错误）
```java
private String nextRoute(UserEntity user) {
    if ("ADMIN".equalsIgnoreCase(user.getRoleCode())) {
        return "/pages/admin-rates/index";
    }
    if ("AGENT".equalsIgnoreCase(user.getRoleCode())) {
        return "/pages/support-chat-v2/index";
    }
    return "/pages/support-chat-v2/index";  // ❌ 错误：普通用户也跳转到客服页面
}
```

### 修复后（正确）
```java
private String nextRoute(UserEntity user) {
    if ("ADMIN".equalsIgnoreCase(user.getRoleCode())) {
        return "/pages/admin-rates/index";
    }
    if ("AGENT".equalsIgnoreCase(user.getRoleCode())) {
        return "/pages/support-chat-v2/index";
    }
    return "/pages/home/index";  // ✅ 正确：普通用户跳转到主页
}
```
