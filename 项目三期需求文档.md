# CardBrother 项目三期需求文档

版本：v0.3 草案  
来源：2026-06-24 与客户沟通确认的三期功能点  
用途：用于确认三期开发范围、角色权限、排期拆分和验收标准。

## 1. 三期目标

三期重点围绕“客服工作台效率、群发运营能力、卖卡沟通链路、余额统计、聊天体验细节”继续增强。目标是让客服和管理员更像正式运营后台一样工作，让用户卖卡、发图、收消息提醒的流程更接近常见即时通讯软件。

核心目标：

1. 聊天页面增加用户列表，客服可快速切换自己负责的客户会话。
2. 管理员和客服都可以群发消息，其中客服只能给自己名下客户群发，管理员可以全员群发。
3. 用户卖卡信息不只是生成订单，也要可编辑成聊天消息发送给专属客服。
4. 客服可以看到自己名下用户余额总量，管理员可以看到全平台用户余额总量。
5. 复制图片粘贴到聊天框后，先进入待发送状态，必须点击发送按钮后才真正发送。
6. 用户发来新消息时，客服端需要有消息提醒音效。

## 2. 角色范围

| 角色 | 三期新增能力 |
| --- | --- |
| 普通用户 | 卖卡时可生成可编辑消息，确认后发送给专属客服；复制图片粘贴后可预览并点击发送；收到客服消息保持原聊天体验 |
| 客服 | 聊天页查看自己客户列表；切换客户会话；给自己客户群发；查看自己负责客户余额总量；收到用户新消息时播放提示音 |
| 管理员 | 给全体用户群发；查看所有用户余额总量；查看群发记录和投递范围 |

## 3. 功能需求明细

### 3.1 聊天页面增加用户列表

功能描述：  
客服进入聊天页面后，需要在聊天界面看到自己负责的用户列表，并能快速切换不同客户会话。

功能规则：

1. 普通用户进入聊天页时，仍然只看到自己和专属客服的会话。
2. 客服进入聊天页时，页面展示自己名下客户列表。
3. 用户列表至少展示：客户用户名、最近一条消息、未读数、最近消息时间。
4. 点击用户列表中的客户后，右侧或当前区域切换为该客户会话。
5. 有未读消息的客户需要明显标识。
6. 客服只能看到分配给自己的客户，不能看到其他客服的客户。
7. 管理员如进入会话管理页，可以查看全部客户会话，但与客服视图区分。

验收标准：

1. 客服登录后能看到自己客户列表。
2. 点击不同客户后，聊天记录正确切换。
3. 新消息到达时，对应客户未读数增加。
4. 客服无法访问非自己分配客户的会话。

### 3.2 管理员和客服群发消息

功能描述：  
管理员和客服都可以群发消息，但群发范围必须受角色权限限制。

功能规则：

1. 客服可以群发给自己名下全部客户。
2. 客服不能群发给其他客服客户，也不能全员群发。
3. 管理员可以给全部普通用户群发。
4. 群发消息进入客户与客服的聊天会话中，作为普通聊天消息展示。
5. 群发支持文本优先，后续可扩展图片、语音、链接、GIF。
6. 群发需要记录发送人、角色、发送范围、消息类型、内容、投递数量、发送时间。

验收标准：

1. 客服群发后，只有自己名下客户收到消息。
2. 管理员群发后，所有活跃普通用户收到消息。
3. 客服尝试全员群发时系统拒绝。
4. 管理员和客服可以查看自己的群发记录；管理员可以查看全部群发记录。

### 3.3 用户卖卡编辑成消息发送给客服

功能描述：  
用户在卖卡流程中填写卡片信息后，系统需要生成一段可编辑的聊天消息，用户确认后发送给专属客服。

功能规则：

1. 用户填写卖卡信息后，系统生成消息草稿。
2. 消息草稿内容包括：卡种、国家/地区、面额、数量、类型、速度、预计结算金额、备注、图片/凭证。
3. 用户可以在发送前编辑这段消息。
4. 用户点击发送后，消息进入专属客服聊天会话。
5. 如卖卡流程仍生成订单，则订单和聊天消息需要关联。
6. 客服收到后，可以直接在聊天中跟进。

验收标准：

1. 用户提交卖卡信息后，能看到自动生成的消息草稿。
2. 修改草稿后发送，客服收到的是修改后的内容。
3. 消息中关键卖卡字段完整展示。
4. 客服端无需刷新即可收到卖卡消息。

### 3.4 用户余额总量统计

功能描述：  
客服和管理员需要看到与权限匹配的用户余额统计，用于运营和结算查看。

功能规则：

1. 系统需要维护用户余额字段或余额计算口径。
2. 客服端展示自己名下所有客户的余额总量。
3. 管理员端展示所有普通用户的余额总量。
4. 统计结果需要排除被封禁、无效或测试用户，具体口径可配置。
5. 如用户余额由交易、提现、卖卡订单计算得出，需要明确余额变动流水。

验收标准：

1. 客服登录后能看到“我的客户余额总量”。
2. 管理员登录后能看到“全平台用户余额总量”。
3. 客服看到的数据只包含自己名下客户。
4. 管理员看到的数据包含全部有效普通用户。

### 3.5 粘贴图片后点击发送再发出

功能描述：  
用户或客服复制图片粘贴到聊天页面后，图片不能立即发送，需要先进入待发送/预览状态，由用户点击发送后才真正上传并发送。

功能规则：

1. 粘贴图片后，输入区显示图片预览。
2. 图片预览旁提供删除/取消按钮。
3. 用户点击发送后，系统上传图片并发送图片消息。
4. 如果用户没有点击发送，图片不进入聊天记录，也不上传到服务器。
5. 支持重新粘贴替换或追加图片，具体多图数量可限制。
6. 图片上传失败时，需要提示失败并保留待发送状态，方便重试。

验收标准：

1. 粘贴图片后不会自动发送。
2. 点击发送后才出现图片消息。
3. 点击取消后图片预览消失，聊天记录不新增图片。
4. 上传失败时有明确提示。

### 3.6 新消息提醒音效

功能描述：  
当用户给客服发送新消息时，客服端需要播放提示音，帮助客服及时响应。

功能规则：

1. 客服在线并打开系统时，收到用户新消息播放提示音。
2. 当前正在查看该客户会话时，可以播放较轻提示音或不播放，具体可配置。
3. 消息来自客服自己时不播放提示音。
4. 群发消息、系统消息是否播放提示音需要区分，默认用户主动消息播放。
5. 浏览器或 App 禁止自动播放时，需要在用户首次交互后启用音效。
6. 客服可以开关消息提示音。

验收标准：

1. 用户发送消息后，客服端无需刷新即可收到消息并播放音效。
2. 客服自己发送消息不会触发音效。
3. 关闭提示音后不再播放。
4. 页面刷新后提示音设置保持生效。

## 4. 权限与安全要求

1. 客服只能查看、群发、统计自己名下客户。
2. 管理员可以查看所有客户、所有客服和全平台统计。
3. 普通用户只能查看自己的会话、余额和卖卡记录。
4. 群发接口必须做后端权限校验，不能只依赖前端隐藏按钮。
5. 聊天消息、卖卡消息、图片消息都要记录发送人和发送时间。

## 8. 2026-06-29 补充确认：客服工作台与账务要求

### 8.1 客服工作台补充范围

1. 客服登录后进入客服工作台，不能只看到聊天窗口，还需要在同一页面看到客户相关信息。
2. 客服可以查看当前选中客户的基础资料、备注、余额、订单、提现、贷款和视频通话记录。
3. 客服可以给客户修改备注，备注用于客服侧快速识别客户，不覆盖客户真实用户名。
4. 客服可以处理自己负责客户的订单；非负责客服不能越权处理。
5. 客服列表在线状态必须表示真实在线状态，离线客户显示灰色状态点，不能把账号正常状态当作在线。
6. 客户订单数量较多时，右侧资料面板使用订单下拉选择查看单个订单详情，避免订单列表过长影响工作台可读性。
7. 客服可切换工作台语言；只翻译系统 UI、状态、按钮和空状态，不翻译用户名、订单号、金额、卡种、聊天消息等业务原始内容。

### 8.2 客服账务视图

功能描述：
客服需要直观看到自己负责客户的账务情况，管理员可看到全平台普通用户账务情况。

功能规则：

1. 客服账务总览展示：客户数、可用余额合计、待结算合计、已提现合计。
2. 客服账务明细展示：客户、可用余额、待结算、已提现、订单数量、待处理订单数量、提现数量、最近更新时间。
3. 客服只能查看自己负责客户的账务明细。
4. 管理员可以查看全部有效普通用户的账务明细。
5. 普通用户不能访问客服账务接口。
6. 账务金额当前按交易与提现实时计算：
   - 可用余额 = 已完成订单金额 - 已完成提现金额，最低为 0。
   - 待结算 = 待处理和处理中的订单金额。
   - 已提现 = 已完成提现金额。
7. 客服处理订单或提现状态后，账务总览和客户明细需要刷新。

验收标准：

1. 客服登录后能在客服工作台看到自己负责客户的账务总览。
2. 客服能看到客户级余额明细，并能点击客户行切换到对应客户会话。
3. 客服处理订单状态后，客户资料和账务视图同步更新。
4. 其他客服不能通过接口查看非自己负责客户的账务数据。
5. 管理员访问账务接口时返回全平台有效普通用户数据。

### 8.3 登录跳转补充

1. 普通用户登录或注册成功后默认进入用户聊天页面 `/pages/support/index`。
2. 客服登录后进入客服工作台 `/pages/support-chat-v2/index`。
3. 管理员登录后进入管理页面 `/pages/admin-rates/index`。
