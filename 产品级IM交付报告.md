# 产品级 IM 交付报告

更新日期：2026-06-30

## 交付结论

本次已把项目中的客服 1 对 1 即时通讯从“基础可用”补强到可交付的产品级闭环：消息有客户端幂等、服务端顺序号、断线游标同步、已读 ACK、WebSocket 实时投递、Redis 多实例广播、腾讯云 Chat 服务端镜像、移动端推送设备绑定、运营重试入口和基础限流。

当前服务端业务消息账本仍以本系统数据库为准，腾讯云用于移动端推送和后续 SDK 级投递增强。这样可以避免把订单、客服、用户关系等业务状态完全绑定到第三方，同时满足后续 iOS/Android 原生推送接入。

## 已完成闭环

### 1. 消息可靠性

- 客户端发送支持 `clientMessageId`，网络重试不会重复落库。
- 服务端写入 `serverSeq`，用于稳定排序和断线续拉。
- 消息状态支持 `PENDING`、`SENT`、`FAILED` 等前端展示和重试。
- 客服消息和好友直聊消息均已覆盖同一套基础协议。

### 2. 断线恢复和已读同步

- 新增按 `serverSeq` 游标同步的消息接口。
- 前端本地缓存会记录会话游标，重连后从最后位置续拉。
- 已读 ACK 会回传 `readSeq`，前端可把本人已发消息标记为已读。
- 旧的按 `afterId` 拉取接口保留，降低升级风险。

### 3. 实时投递和横向扩展

- WebSocket 仍负责 H5、管理端和 App 内实时消息。
- 新增 Redis pub/sub 广播，多个后端实例部署时可以跨节点分发消息。
- Docker Compose 已加入 Redis，并配置健康检查、数据卷和后端依赖。

### 4. 腾讯云 Chat 对接

- 用户会生成稳定 Tencent UserID，并写回本地用户表。
- 服务端支持腾讯云账号导入。
- 支持通过腾讯云 REST API 镜像 1 对 1 文本消息。
- 镜像状态会落库：跳过、待处理、成功、失败、失败原因、腾讯返回消息 key。
- 腾讯云未配置或关闭时，本地 IM 不受影响。

### 5. iOS/Android 原生推送准备

- 后端新增推送设备注册、列表和禁用接口。
- App 启动时会尝试通过 uni-app App-Plus 获取推送客户端信息并上报。
- 推送点击 payload 已约定跳转客服会话页面。
- 已补充《移动端推送接入说明》，说明腾讯 TIMPush、APNs、Android 厂商通道和真机验收项。

### 6. 运营和交付保障

- 新增基础发送限流，防止单用户短时间刷屏拖垮客服会话。
- 新增管理员腾讯云镜像失败列表和重试接口。
- 新增《IM运营交付检查清单》，覆盖 Redis、WebSocket、腾讯云、移动推送、数据备份和异常恢复。

## 三个角色可实现情况

### 用户

- 可发起客服 1 对 1 会话。
- 可发送文本、图片、语音等既有消息类型。
- 网络异常后可重试发送，恢复后可续拉漏掉的消息。
- App 端具备原生推送设备绑定和点击回到会话的代码路径。

### 客服

- 可查看和接入用户客服会话。
- 可实时收发消息。
- 可通过已读同步看到消息状态变化。
- 多客服/管理端部署时，Redis 广播可以支撑多后端实例。

### 管理员/运营

- 可通过新增接口查看腾讯云镜像失败消息。
- 可重试失败镜像。
- 有交付检查清单指导每日检查、上线前检查和异常恢复。
- 可通过环境变量启停腾讯云能力，支持先本地上线再逐步打开第三方能力。

## 需要客户侧配合验收的项目

以下事项需要真实生产资料或设备，代码已预留并接通，但不能在本地环境伪造为已完成：

- 腾讯云 Chat `SDKAppID`、密钥、管理员账号和 UserSig 策略。
- 腾讯 TIMPush 服务开通。
- iOS APNs 证书或 token key 配置。
- Android 各厂商推送通道配置。
- HBuilderX/uni-app 原生打包配置和真机安装包。
- iOS/Android 真实设备锁屏、后台、杀进程状态下的推送到达测试。

## 上线建议

1. 先以本地数据库 IM 为主链路上线，保持 `APP_TENCENT_CHAT_ENABLED=false`。
2. 生产 Redis、MySQL、后端、前端部署稳定后，打开腾讯云沙箱或灰度账号测试镜像。
3. 真机 App 包完成后，按《移动端推送接入说明》完成 iOS/Android 推送验收。
4. 首批客户上线阶段每天检查《IM运营交付检查清单》中的消息积压、镜像失败、WebSocket 连接和 Redis 健康。
5. 客户量接近 1w+ 时，建议把当前内存限流升级为 Redis 全局限流，并接入 Prometheus/Grafana 或腾讯云监控。
