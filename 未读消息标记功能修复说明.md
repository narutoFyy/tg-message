# 未读消息标记功能修复

更新时间：2026-06-26  
版本：v2.2

---

## ✅ 修复内容

### 问题描述
- **问题**：点击客户切换会话后，未读消息的红色角标没有清除
- **原因**：新版页面的 `selectCustomer` 函数中缺少调用 `markSupportRead()` 方法
- **影响**：用户已经查看了消息，但红色角标仍然显示

### 修复方案

#### 1. 切换客户时标记已读

```javascript
function selectCustomer(conv: SupportConversationItem) {
  activeConversationId.value = conv.conversationId
  store.setActiveSupportConversation(conv.conversationId)
  showChat.value = true
  enableAudio()
  connectSocket()
  // ✅ 新增：标记消息为已读，清除未读角标
  store.markSupportRead().catch(() => {})
}
```

#### 2. 页面加载时标记已读

```javascript
onShow(() => {
  checkMobile()
  store.bootstrap().then(() => {
    if (store.state.supportConversations.length > 0) {
      activeConversationId.value = store.state.supportConversations[0].conversationId
      store.setActiveSupportConversation(activeConversationId.value)
      connectSocket()
      // ✅ 新增：页面加载时标记第一个客户的消息为已读
      store.markSupportRead().catch(() => {})
    }
  })
})
```

## 🎯 功能说明

### 未读消息标记逻辑

1. **显示未读角标**
   - 客户发送新消息后，红色角标自动显示
   - 角标上的数字表示未读消息数量

2. **清除未读角标**
   - 点击客户切换到该会话时，自动标记为已读
   - 页面加载时，当前选中的客户会话自动标记为已读
   - WebSocket收到新消息后，如果正在查看该客户，自动标记为已读

3. **后端API**
   - `POST /api/support/conversations/{conversationId}/read`
   - 将该会话的所有未读消息标记为已读

## 🐛 已知问题和注意事项

### 网络延迟
- 标记已读需要调用后端API
- 如果网络延迟较高，角标可能延迟1-2秒消失
- 失败时会静默处理，不影响用户操作

### 多端同步
- 如果同一客服在多个设备登录
- 一端标记已读后，其他端需要刷新才能看到更新
- 未来可以通过WebSocket推送实现多端实时同步

### 页面刷新
- 刷新页面后会重新加载会话列表
- 后端返回的 unreadCount 是最新的准确值
- 前端会根据后端数据显示角标
