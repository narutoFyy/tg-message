## 3. 权限测试

状态：核心接口部分具备，WebSocket 需补强

已具备：
- 用户只能查询自己的客服会话。
- 客服只能查询分配给自己的客户会话。
- 管理员可以查询全部客服会话。
- 生产 profile 关闭 demo fallback。

风险：
- WebSocket 握手只校验 JWT，没有校验用户是否有权进入对应 `channelId`。
- 当前 `WebSocketConfig` 使用 `.setAllowedOriginPatterns("*")`，生产应收紧。
- 需要补越权自动化测试：用户 A 访问用户 B 会话、客服 A 访问客服 B 会话、普通用户访问管理后台。

待办：
- [P0] WebSocket 握手阶段校验 channel 访问权限。
- [P0] 生产环境收紧 WebSocket Origin。
- [P0] 增加权限测试用例覆盖用户/客服/管理员三类角色。
- [P1] 管理员操作增加审计日志。

## 4. 文件与图片上传安全

状态：部分具备

已具备：
- Spring multipart 限制 10MB。
- Nginx `client_max_body_size` 限制 20MB。
- 图片 MIME 白名单：png/jpeg/webp/gif。
- 语音 MIME 白名单。
- 上传文件重命名为 UUID，避免直接使用原文件名落盘。

缺口：
- 只信任 `MultipartFile.getContentType()`，缺少真实文件内容嗅探。
- 没有病毒/恶意文件扫描。
- 上传文件现在通过本地磁盘公开路径访问，不是对象存储私有权限。
- 没有图片压缩、尺寸限制、EXIF 清理。
- 没有上传频率限制。

待办：
- [P0] 使用 Apache Tika 或等价方式做文件内容嗅探。
- [P0] 限制图片尺寸、语音时长和单用户上传频率。
- [P0] 接对象存储，例如 COS/S3/OSS，并使用私有桶 + 临时签名 URL。
- [P1] 接入杀毒扫描或云厂商内容安全。
- [P1] 图片压缩和 EXIF 清理。
