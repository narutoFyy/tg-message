# CardBrother 项目二期客户验收测试与使用说明

版本：v1.0  
用途：用于客户验收、演示讲解、内部测试和交付说明。  
适用范围：当前二期已完成并可演示的功能。

## 1. 项目说明

本项目二期当前主要围绕以下能力进行验收：

1. 用户注册/登录后自动进入专属客服聊天。
2. 系统自动给用户分配专属客服。
3. 首页卖卡入口、卖卡表单、自动生成订单。
4. 提现申请、贷款申请、排行榜、群发、后台管理。
5. 用户只对接自己的专属客服，不开放用户互加好友。

## 4. 客户使用说明

### 4.1 普通用户如何使用

1. 打开首页或登录页。
2. 注册账号或使用已有账号登录。
3. 登录成功后，系统会自动跳转到专属客服聊天页。
4. 用户可在首页查看汇率卡片，并点击 `Sell` 进入卖卡页。
5. 用户可在钱包页发起提现。
6. 用户可在 `Orders` 页面查看自己的订单记录。
7. 用户可在排行榜页查看销售榜和邀请榜。
8. 用户可在贷款页提交贷款申请。

### 4.2 客服如何使用

1. 使用客服账号登录。
2. 登录成功后进入客服聊天页。
3. 客服可通过顶部客户切换入口查看自己名下不同客户会话。
4. 客服可在后台管理页查看订单、提现、贷款等记录。
5. 客服可处理客户消息、跟进卖卡订单、跟进提现。

### 4.3 管理员如何使用

1. 使用管理员账号进入后台登录页。
2. 登录后可进入汇率管理页和管理后台。
3. 管理后台支持查看：
   - 用户列表
   - 客服列表
   - 客服会话
   - 群发记录
   - 订单
   - 提现
   - 贷款
   - 通知提醒
4. 管理员可调整客服状态、查看客户分配关系、处理后台数据。

## 5. 客户验收测试流程

### 5.1 注册/登录自动进入专属客服

验收目标：

- 用户注册成功后自动进入专属客服聊天页
- 用户登录成功后自动进入专属客服聊天页
- 新用户会自动分配到 5 个客服之一

测试步骤：

1. 打开注册页。
2. 输入用户名、邮箱、密码，完成注册。
3. 观察是否自动跳转到聊天页。
4. 查看聊天页是否出现系统欢迎消息。
5. 管理员登录后台，查看该用户是否已经分配专属客服。

通过标准：

- 注册后页面自动跳转到 `support` 聊天页
- 后台能看到该用户的专属客服
- 聊天中能看到系统消息，例如 `User xxx registered.`

### 5.2 用户发送消息，客服是否收到

验收目标：

- 用户给专属客服发消息后，专属客服能收到

测试步骤：

1. 使用普通用户登录。
2. 在专属客服聊天页输入一条测试消息。
3. 使用对应客服账号登录。
4. 打开对应客户会话。
5. 查看是否收到该消息。

通过标准：

- 用户发送后消息出现在聊天页
- 客服账号能看到该消息

### 5.3 首页卖卡流程

验收目标：

- 首页可以进入卖卡页
- 提交卖卡后生成订单
- 订单自动发给专属客服

测试步骤：

1. 用户登录后进入首页。
2. 在任意汇率卡片上点击 `Sell`。
3. 进入 `Sell card` 页面。
4. 选择卡片、国家、金额、数量、类型、速度。
5. 填写卡密/备注，上传图片。
6. 点击 `Confirm`。
7. 观察是否自动跳回专属客服聊天页。
8. 查看聊天中是否出现订单摘要。
9. 管理员进入后台 `Orders` 查看是否生成新订单。

通过标准：

- 卖卡页能正常提交
- 提交后生成订单号
- 专属客服聊天中出现订单消息
- 后台 `Orders` 中出现对应订单记录

### 5.4 提现流程

验收目标：

- 用户提交提现后生成提现记录，并发给专属客服

测试步骤：

1. 用户进入钱包页。
2. 点击 `Withdraw`。
3. 填写金额、国家、姓名、银行、账号、联系方式。
4. 点击提交。
5. 观察是否自动回到专属客服聊天页。
6. 管理员后台进入 `Withdraw` 查看记录。

通过标准：

- 提现申请成功
- 聊天中出现提现申请消息
- 后台出现提现记录

### 5.5 贷款流程

验收目标：

- 用户可提交贷款申请
- 管理员可审核

测试步骤：

1. 用户进入贷款页。
2. 填写贷款金额、国家、用途、联系方式、还款计划。
3. 提交贷款申请。
4. 管理员进入后台 `Loans` 查看申请。
5. 管理员点击 `Approve` 或 `Reject`。
6. 用户回到聊天页查看是否收到结果消息。

通过标准：

- 用户能提交贷款申请
- 后台能看到贷款记录
- 管理员能审核
- 用户收到审核结果

### 5.6 管理员后台验收

验收目标：

- 管理员可以查看和处理二期核心数据

测试步骤：

1. 管理员登录后台。
2. 依次检查以下页面/标签：
   - `Users`
   - `Agents`
   - `Support`
   - `Broadcast`
   - `Orders`
   - `Withdraw`
   - `Loans`
   - `Alerts`
3. 检查是否存在新注册用户、新订单、新提现、新贷款记录。

通过标准：

- 后台所有核心标签可打开
- 数据能正常加载
- 订单、提现、贷款、通知都能看到

## 6. 当前功能说明

### 6.1 Orders 是什么

`Orders` 就是订单页。  
主要用于查看：

- 卖卡订单
- 交易记录
- 当前订单状态

### 6.2 Chat 是什么

`Chat` 是专属客服聊天页。  
普通用户进入后只和自己的专属客服沟通。  
客服进入后可以查看自己名下客户会话。

### 6.3 Message 勾选说明

当前聊天页消息状态规则：

- `✓`：发送成功
- `✓✓`：对方已读

## 7. 当前未纳入客户正式验收的功能

以下功能目前不建议按“已交付”向客户承诺：

1. 实时视频通话正式能力  
   当前只有技术预留和页面入口，正式打视频需要接入第三方 SDK，例如腾讯云 TRTC。

2. 视频通话记录后台  
   需要在接入真实视频 SDK 后补充。

3. 阶梯汇率完整后台配置  
   当前是基础汇率与基础算价能力，复杂的面额区间配置仍建议作为后续增强项。

4. 表情包完整面板  
   当前支持 GIF/图片等基础消息能力，但不是完整 IM 表情系统。

## 8. 客户验收建议顺序

建议客户按下面顺序验收，最容易看出项目价值：

1. 注册/登录是否自动进入专属客服
2. 用户发消息，客服是否收到
3. 首页卖卡 -> 生成订单 -> 客服收到
4. 提现 -> 聊天通知 -> 后台记录
5. 贷款申请 -> 后台审核 -> 用户收到结果
6. 后台查看用户/订单/提现/贷款/通知

## 9. 当前结论

当前二期已经具备一套完整的“用户只对接专属客服”的主业务链路，可用于：

- 用户注册登录演示
- 卖卡流程演示
- 提现流程演示
- 贷款流程演示
- 后台管理演示

如果客户本次重点关注“卖卡 + 专属客服 + 提现 + 后台管理”，当前版本已经可以作为正式验收基础。
