# CardBrother 项目二期需求文档

版本：v0.2 确认版  
来源：客户二期沟通记录、参考截图、需求确认回复  
用途：用于确认二期功能范围、开发排期、报价边界和最终验收标准。

## 1. 二期目标

二期在一期“注册登录、汇率展示、客服聊天、后台管理”的基础上，重点增强卖卡转化、客服运营、用户留存和移动端沟通能力。

核心目标：

1. 所有用户注册或登录后，永久自动进入专属客服聊天页。
2. 系统为客户自动分配专属客服，客服数量暂按 5 个配置。
3. 首页汇率卡片增加出售入口，用户可填写卡面信息，系统自动算价并生成订单。
4. 出售订单、登录提醒、注册提醒、提现申请都能通知专属客服和管理员。
5. 客服可给自己的专属客户群发，管理员可给所有客户群发。
6. 支持印度、喀麦隆、加纳多国家汇率和结算币种。
7. 用户可提交提现，提交后立即扣除余额并通知专属客服手动转账结款。
8. 增加排行榜、贷款、表情包、粘贴图片发送、手机端实时视频通话等能力。

## 6. 功能需求明细

### 6.1 专属客服自动分配

功能描述：每个客户都绑定一个专属客服，用户注册或登录后进入该客服聊天。

功能规则：

1. 系统暂按 5 个客服账号运行。
2. 新用户注册成功后，系统自动分配专属客服。
3. 老用户如果没有专属客服，下一次登录时自动补充分配。
4. 分配规则为随机平均分配：优先从当前客户数较少的客服中随机选择，尽量让 5 个客服客户数量接近。
5. 用户每次登录或注册成功后都自动跳转专属客服聊天页，永久生效。
6. 管理员后台可查看和调整客户的专属客服。

验收标准：

1. 新用户注册后自动进入客服聊天页。
2. 用户登录后自动进入客服聊天页。
3. 多个新用户注册后，5 个客服分配数量大致平均。
4. 客服只能看到自己名下客户会话。
5. 管理员可看到全部客服分配关系。

### 6.2 用户登录 / 注册提醒

功能描述：用户登录或注册后，专属客服和管理员收到提醒，并可以主动聊天。

功能规则：

1. 用户注册成功时，专属客服收到站内提醒。
2. 用户登录成功时，专属客服收到站内提醒。
3. 管理员收到全局用户注册/登录站内提醒。
4. 同时在聊天会话内插入系统消息，例如“用户 Jay 已登录”。
5. 提醒内容包含用户名、手机号/邮箱、专属客服、时间、事件类型。
6. 客服或管理员点击提醒后进入对应聊天会话。

验收标准：

1. 用户注册后，客服端和管理员端出现站内提醒。
2. 用户登录后，客服端和管理员端出现站内提醒。
3. 对应聊天会话里出现系统消息。
4. 客服/管理员可以主动给用户发送消息。

### 6.3 首页出售按钮

功能描述：首页汇率卡片新增 `出售` 按钮，用户点击后进入卖卡填写页。

功能规则：

1. 每个汇率卡片展示 `出售` 按钮。
2. 点击后进入 `Sell card` 页面。
3. 页面默认带入当前卡片名称、卡片图标、当前国家、当前汇率。
4. 用户可以点击 `Change` 更换卡片。
5. 如果用户切换首页国家，出售页使用该国家对应汇率。

验收标准：

1. 首页每个卡片都能点击出售。
2. 进入出售页后卡片信息正确带入。
3. 点击 `Change` 可切换其他卡片。

### 6.4 卖卡填写页

功能描述：按参考图实现完整卖卡表单，用户填写卡片信息并提交。

页面字段：

| 字段 | 说明 |
| --- | --- |
| Card / Category | 卡片类型，例如 Razer Gold、Steam、Apple |
| Country | 卡片国家/币种，例如 AUD；同时要关联用户当前结算国家 |
| Balance | 卡面额，支持手动输入 |
| 快捷金额 | 50、100、200、500 等快捷按钮 |
| Quantity | 数量，支持加减 |
| Type | Physical、Code、Horizontal Image、Whiteboard 等 |
| Speed | Fast、Slow |
| Card Data | 卡密/备注，可选 |
| Upload Photo | 上传卡片图片，可多图 |
| Settlement Amount | 系统自动计算结算金额 |
| Confirm | 确认出售 |

算价规则：

1. 默认公式：`结算金额 = 卡面额 x 数量 x 当前汇率`。
2. 后台支持阶梯汇率：管理员可按卡片、国家、面额区间配置不同汇率。
3. 低面额可配置更高汇率，例如 50 美元使用一档，100 美元使用另一档。
4. 页面金额变化时实时刷新 Settlement Amount。

提交规则：

1. 用户点击 `Confirm` 后生成出售订单。
2. 订单状态默认为待客服处理。
3. 系统将订单摘要作为聊天消息发送给专属客服。
4. 用户提交后自动跳转到专属客服聊天页。

验收标准：

1. 用户填写 50 美元卡片后，系统自动计算结算金额。
2. 用户切换数量、国家、卡类型后，金额重新计算。
3. 上传图片后可提交订单。
4. 提交后生成订单，并在客服聊天中出现订单信息。
5. 客服端可看到该订单并继续处理。

### 6.5 出售订单管理

功能描述：用户确认出售后必须生成订单，便于客服和管理员跟进。

订单字段：

| 字段 | 说明 |
| --- | --- |
| 订单号 | 系统生成 |
| 用户 | 提交订单的客户 |
| 专属客服 | 当前绑定客服 |
| 卡片名称 | 用户选择的卡片 |
| 卡片国家/币种 | 用户选择的卡国家 |
| 结算国家 | 用户当前国家，例如印度、喀麦隆、加纳 |
| 卡面额 | 用户填写 |
| 数量 | 用户填写 |
| 汇率 | 下单时锁定 |
| 结算金额 | 系统计算 |
| 类型 | Physical、Code 等 |
| 速度 | Fast、Slow |
| 卡密/备注 | 用户填写 |
| 图片 | 用户上传 |
| 状态 | 待处理、处理中、已完成、已取消 |

验收标准：

1. 用户提交出售后后台出现订单。
2. 订单保留下单时汇率，不受后续汇率修改影响。
3. 客服可按自己客户查看订单。
4. 管理员可查看所有订单。

### 6.6 群发消息

功能描述：客服可向自己的专属客户群发，管理员可向所有客户群发。

功能规则：

1. 客服只能群发自己名下客户。
2. 管理员可一键群发所有普通客户。
3. 群发消息支持文字、图片、语音、链接、表情包。
4. 群发后消息进入对应聊天会话。
5. 系统记录发送人、发送时间、目标人数、成功数、失败数、消息类型。

验收标准：

1. 客服群发后，只有该客服名下客户收到消息。
2. 管理员群发后，所有普通用户都收到消息。
3. 图片、语音、链接、表情包均可群发。
4. 后台可查看群发记录。

### 6.7 用户查看专属客服

功能描述：用户只对接自己的专属客服，不开放用户之间互加好友，也不开放用户与管理员单独聊天。

功能规则：

1. 用户页面展示专属客服名称、头像、在线状态。
2. 用户点击专属客服进入客服聊天。
3. 管理员负责后台查看消息、提醒、订单、提现、贷款等记录，不作为前台独立聊天对象展示给用户。

验收标准：

1. 用户能看到专属客服信息。
2. 用户可给专属客服发消息。
3. 用户与客服沟通内容在后台可被管理员查看和管理。

### 6.8 多国家汇率

功能描述：新增印度、喀麦隆、加纳。用户切换国家后，首页美元汇率自动切换。

国家范围：

| 国家 | 结算币种 | 说明 |
| --- | --- | --- |
| 印度 | INR | 印度卢比 |
| 喀麦隆 | XAF | 中非法郎 |
| 加纳 | GHS | 加纳塞地 |

功能规则：

1. 首页国家切换支持印度、喀麦隆、加纳。
2. 后台汇率管理支持按国家维护汇率。
3. 同一张卡可在不同国家配置不同汇率。
4. 用户切换国家后，首页只显示该国家对应汇率。
5. 卖卡自动算价使用当前国家对应汇率。
6. 卖卡页面的卡片国家/币种与首页结算国家要分别保存，避免混淆。

验收标准：

1. 管理员可分别维护 INR、XAF、GHS 汇率。
2. 用户切换国家后首页汇率变化。
3. 同一张卡在不同国家下单，结算金额不同。

### 6.9 提现功能

功能描述：用户在余额旁点击提现，填写银行资料后提交，系统立即扣除余额并通知专属客服手动转账结款。

功能规则：

1. 用户余额区域新增 `提现` 按钮。
2. 用户每次提现都填写银行账户资料。
3. 提交前校验余额是否充足。
4. 提交成功后立即扣除用户余额。
5. 生成提现记录，状态默认为待客服转账。
6. 系统把提现信息发送到用户与专属客服的聊天会话。
7. 客服根据提现资料手动转账结款。
8. 暂时不做驳回退款流程。
9. 管理员可查看全部提现记录。

提现字段：

| 字段 | 说明 |
| --- | --- |
| 提现金额 | 用户输入 |
| 国家 | 用户选择 |
| 收款人姓名 | 用户填写 |
| 银行名称 | 用户填写 |
| 银行卡号 / 收款账号 | 用户填写 |
| 联系方式 | 默认带用户手机号，可修改 |
| 备注 | 可选 |
| 状态 | 待客服转账、已完成 |

验收标准：

1. 用户余额旁可以看到提现按钮。
2. 用户填写银行资料并提交后，余额立即减少。
3. 专属客服聊天中收到提现申请消息。
4. 后台生成提现记录。
5. 客服可标记提现已完成。

### 6.10 排行榜

功能描述：新增排行榜页面，参考图实现销售榜和邀请榜。

页面规则：

1. 排行榜顶部有 `Sales` 和 `Invitation` 两个切换标签。
2. 支持按月份筛选，例如 Jun, 2026。
3. 支持月榜展示。
4. 前三名使用大卡片展示，包含头像、脱敏用户名、排名、销售额/邀请数、奖励金额。
5. 第 4 名以后使用列表展示。
6. 底部固定展示当前用户自己的排名、头像、名称、当前销售额/邀请数。
7. 提供 `Rewards` 按钮查看奖励。

销售榜规则：

1. 按用户当月已完成出售订单的结算金额排序。
2. 订单未完成时不计入销售榜。
3. 金额展示需按当前结算币种格式化。

邀请榜规则：

1. 按用户当月邀请注册并完成有效行为的人数排序。
2. 有效行为规则需后台可配置，默认可先按“邀请用户成功注册”统计。

奖励规则：

1. 后台可配置每个名次或名次区间的奖励金额。
2. 排行榜页面展示奖励金额。
3. 管理员可查看奖励发放记录。

验收标准：

1. 用户可切换 Sales / Invitation。
2. 用户可按月份查看排行。
3. 前三名样式突出展示。
4. 当前用户即使排名较低，也能在底部看到自己的排名。
5. 管理员可配置排行榜奖励。

### 6.11 贷款功能

功能描述：贷款功能纳入二期，先实现基础申请、审核、记录流程。

建议二期范围：

1. 用户可进入贷款页面。
2. 用户填写贷款金额、期限、用途、联系方式。
3. 系统生成贷款申请记录。
4. 管理员后台可查看贷款申请。
5. 管理员可审核通过或拒绝。
6. 审核结果通过站内提醒和聊天消息通知用户。
7. 贷款申请记录可在用户端查看。

仍需后续确认的细则：

1. 是否需要利息。
2. 是否需要逾期规则。
3. 是否需要自动放款。
4. 是否占用用户余额。
5. 是否需要还款计划和还款记录。

验收标准：

1. 用户可提交贷款申请。
2. 管理员可审核贷款申请。
3. 用户能收到审核结果。
4. 后台能查看贷款记录。

### 6.12 表情包功能

功能描述：聊天支持类似 QQ 的表情包/GIF 发送。

功能规则：

1. 聊天输入框增加 GIF/表情入口。
2. 用户、客服、管理员都可发送表情包。
3. 支持系统内置表情包。
4. 支持 GIF 动图。
5. 表情包可用于普通聊天和群发。
6. 二期先做基础表情面板，收藏/自定义表情可作为扩展。

验收标准：

1. 聊天页可打开表情面板。
2. 点击表情后发送到聊天记录。
3. 对方能正常看到表情/GIF。
4. 群发也可发送表情包。

### 6.13 支持复制粘贴图片发送

功能描述：聊天输入区域支持复制图片后直接粘贴并发送。

功能规则：

1. H5 端支持在聊天输入框粘贴剪贴板图片。
2. 粘贴图片后自动上传图片。
3. 上传成功后作为图片消息发送。
4. 支持客服聊天。
5. 不支持剪贴板图片的设备提示用户使用图片上传按钮。

验收标准：

1. 在电脑浏览器复制截图后，进入聊天页面按 `Ctrl + V`。
2. 聊天中出现图片消息。
3. 对方可以正常查看图片。

### 6.14 手机端实时视频

功能描述：H5、Android、iOS 都要支持实时视频通话。

功能规则：

1. 用户可在专属客服聊天中发起视频通话。
2. 专属客服可接听或拒绝。
3. 通话中支持摄像头和麦克风。
4. 支持挂断。
5. 记录通话开始时间、结束时间、通话状态。
6. 管理员可查看视频通话记录。

费用说明：

1. 视频通话功能本身可以开发，但线上稳定运行通常会产生费用。
2. 如果使用第三方音视频 SDK，例如腾讯云 TRTC、声网 Agora、ZEGO 等，一般会按通话分钟数、带宽或套餐收费，优点是稳定、省开发和运维时间。
3. 如果自建 WebRTC，代码框架本身不一定收费，但生产环境通常仍需要 TURN 中继服务器、带宽、服务器维护费用；跨国家、弱网、移动网络场景下更依赖中继服务。
4. 建议二期采用第三方音视频 SDK，先由客户确认服务商和费用承担方式。如果客户暂时不确定，可先做技术预留和页面入口，正式通话功能在选定 SDK 后接入。

验收标准：

1. H5 端可发起和接听视频。
2. Android App 可发起和接听视频。
3. iOS App 可发起和接听视频。
4. 双方可以看到视频画面并听到声音。
5. 任意一方挂断后通话正常结束。

## 8. 二期核心流程

### 8.1 用户注册 / 登录到专属客服流程

1. 用户注册或登录。
2. 系统检查是否已有专属客服。
3. 没有专属客服则按随机平均分配规则绑定 5 个客服之一。
4. 系统发送站内提醒给专属客服和管理员。
5. 系统在聊天内插入登录/注册系统消息。
6. 用户自动跳转到专属客服聊天页。

### 8.2 用户出售卡片流程

1. 用户进入首页。
2. 用户选择国家。
3. 用户点击汇率卡片上的 `出售`。
4. 进入 Sell card 页面。
5. 用户选择卡片、卡国家、面额、数量、类型、速度，填写卡密/备注并上传图片。
6. 系统根据后台汇率和阶梯规则计算 Settlement Amount。
7. 用户点击 `Confirm`。
8. 系统生成出售订单。
9. 系统把订单摘要发送给专属客服。
10. 用户跳转到专属客服聊天页继续沟通。

### 8.3 提现流程

1. 用户进入余额页面。
2. 点击 `提现`。
3. 填写国家、银行账户、提现金额。
4. 系统校验余额。
5. 提交成功后立即扣余额。
6. 系统生成提现记录。
7. 系统把提现申请发送给专属客服。
8. 客服手动转账结款。
9. 客服标记提现已完成。

### 8.4 群发流程

1. 客服或管理员进入群发页面。
2. 客服选择自己的客户，管理员选择全部客户。
3. 选择消息类型：文字、图片、语音、链接、表情包。
4. 编辑内容并确认发送。
5. 系统发送到对应聊天会话。
6. 后台生成群发记录。

### 8.5 排行榜流程

1. 用户进入排行榜页面。
2. 选择 Sales 或 Invitation。
3. 选择月份。
4. 系统展示前三名、排名列表、当前用户排名。
5. 用户点击 Rewards 查看奖励。
6. 管理员后台配置奖励规则和统计规则。

## 9. 仍需客户最终确认

以下内容已经不影响二期范围成立，但会影响具体开发细节和报价：

1. 视频通话选用哪家 SDK：腾讯云 TRTC、声网 Agora、ZEGO，还是自建 WebRTC。
2. 视频通话费用由客户承担，还是先用测试额度开发演示后再购买正式套餐。
3. 贷款是否需要利息、期限、逾期、还款计划。
4. 排行榜奖励是否真实发放到余额，还是只展示奖励记录。
5. 邀请榜的“有效邀请”标准：注册即有效，还是注册后完成卖卡才有效。
6. 表情包是否允许用户自定义上传，还是只用系统内置。
7. 卖卡图片是否必须上传，还是部分类型可以不上传。

## 10. 二期交付验收口径

二期完成后，应至少满足以下验收结果：

1. 所有用户注册/登录后永久自动跳转专属客服聊天。
2. 5 个客服按随机平均规则自动分配客户。
3. 用户登录/注册后，客服和管理员收到站内提醒，聊天内出现系统消息。
4. 首页每个汇率卡片都有出售入口。
5. Sell card 页面按参考图实现完整表单。
6. 用户填写卡面额后系统自动算出 Settlement Amount。
7. 管理员可配置按国家、卡片、面额区间的阶梯汇率。
8. 用户确认出售后生成订单，并发送给专属客服。
9. 客服可群发自己的专属客户，管理员可群发所有客户，消息类型支持文字、图片、语音、链接、表情包。
10. 用户可看到专属客服和管理员，并可分别聊天。
11. 印度、喀麦隆、加纳切换后首页汇率和卖卡结算自动变化。
12. 用户可提交提现，余额立即扣除，提现信息发送给专属客服。
13. 排行榜支持销售榜、邀请榜、月份筛选、前三名展示、当前用户排名和奖励展示。
14. 用户可提交贷款申请，管理员可审核。
15. 聊天支持表情包/GIF。
16. H5 聊天支持复制粘贴图片直接发送。
17. H5、Android、iOS 支持实时视频通话。
18. 管理员后台可查看分配、订单、提现、群发、排行榜、贷款、视频相关记录。
