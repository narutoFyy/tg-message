# 移动端原生推送接入说明

## 当前代码已完成

- 后端已支持 `/api/push/devices` 注册、查询、注销当前登录用户设备。
- uni-app 启动时会在 App-Plus 环境读取原生 push client id/token 并上报后端；H5 环境自动跳过。
- 推送点击 payload 支持通过 `channelType=support` 和 `channelId=<conversationId>` 跳转到对应客服会话。
- 腾讯 Chat 混合模式已支持本地消息落库后镜像到 Tencent Cloud Chat，失败不会影响本地消息发送。

## 腾讯云/原生侧还需要客户准备

1. 在腾讯云 Chat/TIMPush 控制台启用应用能力。
2. 配置 iOS APNs 证书或 APNs Auth Key，并确认 bundle id 与 `gift-chat-uni/src/manifest.json` 一致。
3. 配置 Android 厂商通道，例如华为、小米、OPPO、vivo、荣耀、FCM，按目标市场选择。
4. 在原生打包环境加入 Tencent TIMPush/uni-app 对应插件或厂商推送配置。
5. 后端生产环境填写：
   - `APP_TENCENT_CHAT_ENABLED=true`
   - `APP_TENCENT_CHAT_SDK_APP_ID`
   - `APP_TENCENT_CHAT_SECRET_KEY` 或 `APP_TENCENT_CHAT_ADMIN_USER_SIG`
   - `APP_TENCENT_CHAT_ADMIN_IDENTIFIER`

## 推送 payload 约定

隐私模式建议只发泛化文案：

```json
{
  "title": "CardBrother",
  "body": "You have a new support message",
  "channelType": "support",
  "channelId": "support-1"
}
```

不建议在锁屏通知中发送卡密、金额、银行卡、订单敏感信息。

## 实机验收矩阵

- iOS 前台、后台、杀进程：收到通知并点击进入正确客服会话。
- Android 前台、后台、杀进程：至少覆盖客户目标市场的主要厂商机型。
- 同一账号多设备登录：每台设备都能注册、收推送、点击路由正确。
- 退出登录后：本地 token 清理，后续版本可进一步调用注销接口关闭旧设备。
