# CardBrother 一期客户验收测试与使用说明

本文档用于客户验收和日常演示操作。测试人员可以按下面步骤逐项点击，确认普通用户、客服、管理员后台功能是否符合一期需求。

## 3. 角色权限说明

| 角色 | 可以做什么 | 不可以做什么 |
| --- | --- | --- |
| 普通用户 | 注册登录、查看汇率、联系客服、添加好友、好友私聊、发图片/语音、拉黑、查看交易入口 | 不能进入后台管理 |

## 4. 普通用户使用与验收步骤

### 4.1 登录

1. 打开 http://127.0.0.1:5174。
2. 在 `Email or Phone` 输入 `demo@cardnova.app` 或 `13800138000`。
3. 在 `Password` 输入 `demo12345`。
4. 点击 `Sign In`。

预期结果：进入首页，页面顶部显示 CardBrother，底部有 `Home / Chat / SELL / Transaction / Me` 导航。

### 4.2 注册新用户

1. 在登录页点击 `Sign up`。
2. 填写 `Username`。
3. `Email` 和 `Phone` 至少填写一个。
4. 填写 `Password`。
5. 点击 `Create account`。

预期结果：注册成功后自动进入客服聊天页，用户可以直接向客服发送消息。

注意：当前注册不做短信验证码校验。如果客户要求手机号验证码注册，需要先提供短信服务商资料。

### 4.3 首页查看汇率

1. 登录普通用户后进入首页。
2. 查看汇率列表，每条汇率包含卡片名称、汇率值、状态、更新时间。
3. 点击顶部分类，例如 `Steam`、`Razer Gold`、`All Card`。

预期结果：点击具体卡片分类后，只显示对应卡片汇率；点击 `All Card` 后恢复显示全部汇率。

### 4.4 联系客服

1. 点击底部 `Chat`。
2. 在输入框输入一条文字消息。
3. 点击 `Send`。
4. 可点击快捷问题，例如 `General questions`、`Order Inquiry`。
5. 可点击 `Image` 发送图片，点击 `Voice` 录制语音。

预期结果：消息会出现在聊天记录中；客服账号登录后可以看到该用户会话并回复。

### 4.5 好友和私聊

1. 进入 `Me`。
2. 点击 `Friends`。
3. 在 `Search username` 输入用户名，例如 `gift_hunter`。
4. 点击 `Add` 发起好友申请。
5. 对方账号登录后进入 `Friends`，在 `Pending requests` 点击 `Accept`。
6. 建立好友后，选择好友，输入文字并点击 `Send`。
7. 可点击 `Image` 发送图片，点击 `Voice` 发送语音。

预期结果：双方成为好友后可以一对一私聊；非好友不能直接私聊。

### 4.6 拉黑和黑名单

1. 进入 `Friends`。
2. 选择一个好友。
3. 点击 `Block`。
4. 回到 `Me`，点击 `Blacklist`。

预期结果：黑名单列表展示被拉黑用户的用户名、手机号、加入黑名单时间；拉黑后双方不能继续发起互动或继续私聊。

### 4.7 交易入口

1. 点击底部 `Transaction`。
2. 查看交易记录列表和状态。
3. 或进入 `Friends`，选择好友，点击 `New Trade`。
4. 填写卡片名称、面值、打款金额、备注，可上传凭证。
5. 点击 `Create Trade`。

预期结果：可以创建和查看交易记录。一期重点是预留交易入口和基础记录，不包含完整支付、托管、争议处理流程。

## 5. 客服使用与验收步骤

### 5.1 客服登录

1. 打开 http://127.0.0.1:5174。
2. 输入客服账号 `support_luna` 或 `luna@cardnova.app`。
3. 输入密码 `demo12345`。
4. 点击 `Sign In`。

预期结果：客服登录后进入客服聊天页，而不是后台管理页。

### 5.2 接待用户会话

1. 客服登录后进入 `Customer Service` 页面。
2. 如果有多个用户会话，页面顶部会显示会话切换条。
3. 点击不同用户会话。
4. 输入回复内容并点击 `Send`。

预期结果：客服只能处理聊天；不能进入汇率管理、用户管理、客服账号管理等后台功能。

## 6. 管理员后台使用与验收步骤

### 6.1 管理员登录

1. 打开 http://127.0.0.1:5174/#/pages/admin-login/index。
2. 在 `Admin account` 输入 `admin_mia` 或 `admin@cardnova.app`。
3. 在 `Password` 输入 `demo12345`。
4. 点击 `Enter Admin`。

预期结果：进入 `Admin Rates` 页面。若使用普通用户或客服账号登录，会提示需要管理员账号，不能进入后台。

### 6.2 新增汇率

1. 管理员进入 `Admin Rates`。
2. 在 `Card Name` 输入卡片名称，例如 `Test Card`。
3. 在 `Region` 输入地区，例如 `NG`。
4. 在 `Rate` 输入汇率，例如 `1$ ≈ ₦999.99`。
5. 点击 `Create Rate`。

预期结果：页面下方 `Rate feed` 出现新汇率，状态默认为启用。

### 6.3 编辑汇率

1. 在 `Rate feed` 找到要修改的汇率。
2. 点击该条记录右侧的 `Edit`。
3. 上方表单会自动带出当前卡片名称、地区、汇率。
4. 修改 `Rate`，例如改为 `1$ ≈ ₦1001.00`。
5. 点击 `Update Rate`。

预期结果：汇率记录更新成功，列表显示新的汇率值和更新时间。

### 6.4 停用 / 启用汇率

1. 在 `Rate feed` 找到目标汇率。
2. 如果当前是启用状态，点击 `Pause`。
3. 如果当前是停用状态，点击 `Activate`。

预期结果：状态会在启用和停用之间切换。普通用户首页仍可看到汇率记录，并能看到 `Active` 或 `Paused` 状态。

### 6.5 删除汇率

1. 在 `Rate feed` 找到目标汇率。
2. 点击 `Delete`。

预期结果：该汇率从后台列表删除；普通用户首页刷新后不再显示这条汇率。

## 7. 汇率修改的端到端验收

这一条用于证明“后台修改汇率后，用户端确实同步显示”。

1. 管理员打开后台登录页并登录 `admin_mia`。
2. 进入 `Admin Rates`。
3. 找到 `Steam`，点击 `Edit`。
4. 把 `Rate` 修改成一个容易识别的值，例如 `1$ ≈ ₦1234.56`。
5. 点击 `Update Rate`。
6. 另开一个浏览器窗口，打开 http://127.0.0.1:5174。
7. 使用普通用户 `demo@cardnova.app / demo12345` 登录。
8. 在首页顶部分类点击 `Steam`。

预期结果：普通用户首页只显示 Steam 汇率，并且汇率值显示为管理员刚才修改的 `1$ ≈ ₦1234.56`，更新时间同步变化。

验收完成后，如果需要恢复演示数据，可由管理员再次进入 `Admin Rates`，把 `Steam` 改回原汇率。

## 8. Admin Console 验收步骤

管理员登录后可通过设置页入口或直接打开 `/#/pages/admin-console/index` 进入后台控制台。

### 8.1 查看用户

1. 进入 `Admin Console`。
2. 点击 `Users`。

预期结果：显示用户列表，包括用户名、手机号、邮箱、角色、状态、注册时间、黑名单状态。

### 8.2 创建和启停客服

1. 点击 `Agents`。
2. 在 `Create support agent` 填写客服用户名、邮箱、手机号、密码。
3. 点击 `Create Agent`。
4. 在客服列表点击 `Disable` 或 `Enable`。

预期结果：可以新增客服账号，并控制客服账号是否启用。

### 8.3 手动分配客服会话

1. 点击 `Support`。
2. 在某个用户会话的 `Agent username` 输入客服用户名，例如 `support_luna`。
3. 点击 `Assign`。

预期结果：会话被分配给指定客服；客服登录后可以在会话列表中看到并接待该用户。

### 8.4 查看客服聊天记录

1. 点击 `Support`。
2. 展开或查看用户会话下方消息列表。

预期结果：管理员能看到客服会话的完整历史消息，包括发送人、时间、消息类型、消息内容。

### 8.5 查看用户私聊记录

1. 点击 `Direct`。
2. 可直接查看已有私聊记录。
3. 在 `Filter by username` 输入用户名，例如 `cardnova_user`。
4. 点击 `Search`。

预期结果：只显示与该用户相关的私聊会话，包含双方用户名、会话状态、消息数量和消息内容。

## 9. 上线前客户需要提供或确认的内容

| 项目 | 是否必须 | 说明 |
| --- | --- | --- |
| 短信服务商 API | 如果要验证码登录则必须 | 需要服务商名称、AccessKey、签名、模板 ID、验证码有效期规则 |
| 正式域名和 HTTPS 证书 | 必须 | 用于 H5/API/WebSocket 正式访问 |
| 生产数据库 | 必须 | 建议使用 MySQL，提供地址、账号、密码、库名 |
| JWT 密钥 | 必须 | 生产环境必须使用客户自己的强密钥 |
| 上传文件存储目录 | 必须 | 用于图片、语音、交易凭证 |
| 正式管理员账号 | 必须 | 上线后替换或禁用演示账号 |
| App 打包资料 | 如需上架 App 则必须 | App 名称、图标、包名、证书、应用商店资料 |
| 备案 / 合规材料 | 按客户地区要求 | 域名、短信、应用上架可能需要 |

## 10. 当前交付结论

一期需求中的注册登录、汇率展示、客服接待、多客服、好友、私聊、图片/语音、黑名单、后台汇率、用户管理、客服管理、会话记录查看均已具备可验收流程。

当前唯一明确依赖客户外部资料的功能是短信验证码登录 / 注册。未提供短信服务商 API 前，系统支持邮箱 / 手机号 + 密码注册登录，但不发送短信验证码。
